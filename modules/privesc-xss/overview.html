# Module: Privilege Escalation via XSS

1. Core Concept 🧠

PrivEsc via XSS is an advanced chaining technique. Instead of just popping an alert box, we use XSS to force an admin user's browser to perform actions on our behalf, or we steal their session cookie. Because the victims browser executes our payload while they are authenticated, their requests to the app are trusted.

Imagine an attacker giving a coffee shop manager a script they unconsciously read, forcing them to hand over the store keys to the attacker.

Attack Steps

  1. Log in to the vulnerable app as a standard user.
  2. Find the "Update Profile Biography" section, which is vulnerable to Stored XSS.
  3. Write a payload that fetches the `/admin/promote` endpoint.
  4. Wait for the automated admin bot to view your profile in the Lab simulator.
  5. Log out, log back in, and verify you are an Administrator.

Payload Example

<script>
  fetch('/api/promote-user', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username: "attacker_account", role: "admin" })
  });
</script>

Fix / Mitigation

In addition to strict Contextual Output Encoding (to prevent XSS), critical actions must require re-authentication or strong CSRF tokens that cannot be easily scraped by JS.
