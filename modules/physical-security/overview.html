# Module: Physical Security

1. Core Concept 🧠

Physical Security is the protection of personnel, hardware, software, networks, and data from physical actions and events that could cause serious loss or damage to an enterprise. It is often the most overlooked aspect of cybersecurity.

Real-world analogy: It doesn't matter if your servers are protected by 2048-bit RSA encryption and three firewalls if an attacker can literally walk into the server room and carry the server out the back door.

🧪 Lab: BadUSB Keystroke Injection

Topic: Hardware Difficulty: Easy Time: 20 mins

What is a BadUSB? Devices like the Hak5 Rubber Ducky look like standard USB flash drives, but the computer recognizes them as keyboards. Since computers implicitly trust keyboards, the device can rapidly "type" malicious commands faster than a human ever could as soon as it is plugged in.

Expected Output

Fix / Mitigation

Block unauthorized USB devices using Endpoint Protection (Device Control policies). Use physically secure server racks, implement mantraps, enforce "No Tailgating" rules with security guards, and train staff to never plug in random found devices.
