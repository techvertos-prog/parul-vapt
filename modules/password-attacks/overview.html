Module: Password Attacks & Cracking

📚 Theory & Class Notes

Password Cracking is the process of recovering passwords from data that has been stored or transmitted by a computer system. It leverages vast dictionaries, rulesets, and GPU computation to guess the plain text of a cryptographic hash.

Vulnerabilities: Weak hashing algorithms (MD5/SHA1), Lack of salt, weak user password policies, and exposed /etc/shadow files.
Tools: Hashcat, John the Ripper, Hydra (online brute-forcing), Medusa.

1. Core Concept 🧠

Password Attacks involve techniques to discover a user's secret credential. They fall into two main categories:

  • Online Attacks (Brute Force / Credential Stuffing): The attacker writes a script that interacts with the live authentication service (e.g., SSH, Web Login). The script tries thousands of passwords per second.
  • Offline Attacks (Hash Cracking): The attacker steals the database containing password hashes. Since hashes are mathematically irreversible, the attacker uses an offline tool to generate hashes of dictionary words until they find a matching hash.

Real-world analogy: Online attacks are like standing at a locked door trying thousands of physical keys (slow, easily caught by security cameras). Offline attacks are like stealing the blueprint of the lock, taking it home to a supercomputer in a bunker, and calculating the exact key shape in peace.

2. The Attack Flow (Offline Hash Cracking) 🎯

👨💻 Attacker Obtains Hashes (e.g. via SQLi)
↓
🔍 Identifies Hash Type Output: $1$ = MD5
↓
⚙️ Supplies Wordlist to Hashcat or John the Ripper
↓
🏁 Software recovers plaintext password offline
💡
What is Rockyou.txt? In 2009, a company named RockYou was hacked, exposing 32 million unencrypted passwords. Security researchers saved this list and it became the gold standard "dictionary" for hash cracking. If you use a password that has ever been caught in a breach, it is in `rockyou.txt` and an attacker can crack it instantly.

Context & Setup

By clicking "Start Lab" above, a lightweight Alpine linux container briefly spins up to generate a mocked `/etc/shadow` hash file dynamically. Follow the steps below using your local Kali/Parrot instance or a cloud VM with `john` installed.

Attack Steps (Offline)

# 1. Inspect the stolen hash file format:
$ cat /tmp/hashes.txt
admin:$1$EliRH15m$M2TaFZ/PhgWrUvRdtPgK8/

#You can generate password hashes using command 
$ openssl passwd -1 YOUR_PASSWORD

# The prefix $1$ tells us this is an MD5Crypt hash.

# 2. Start John the Ripper using the default wordlist
$ john --wordlist=/usr/share/wordlists/rockyou.txt /tmp/hashes.txt

# 3. John runs incredibly fast on CPU (Hashcat scales better on GPU)
# Press Spacebar while it runs to see status updates.

# 4. Once cracked, John stops. View the cracked passwords:
$ john --show /tmp/hashes.txt

Expected Terminal Output

Loaded 1 password hash (md5crypt, crypt(3) $1$ [MD5 256/256 AVX2 8x3])
Will run 4 OpenMP threads
Press 'q' or Ctrl-C to abort, almost any other key for status
password123 (admin)
1g 0:00:00:02 DONE (2026-03-17 14:22) 0.4566g/s 45228p/s 45228c/s 45228C/s password123..
Use the "--show" option to display all of the cracked passwords reliably

$ john --show /tmp/hashes.txt
admin:password123
1 password hash cracked, 0 left

Fix / Mitigation

For Online: Implement Account Lockout Policies and Rate Limiting (e.g., Fail2Ban). Disable SSH Password Auth, use Keys only.
For Offline: Use strong, slow-hashing algorithms (Argon2id, bcrypt, PBKDF2) with high work factors and cryptographic salts. A fast algorithm like MD5 allows an attacker to guess billions of passwords a second; bcrypt slows them down to thousands.
